# Локальный контекст проекта `project-max`

Этот файл задаёт локальные правила для `D:\opencode\project-max`.
Он дополняет корневой `D:\opencode\AGENTS.md` и имеет более высокий приоритет внутри этой папки.

## Назначение проекта

`project-max` — self-hosted система, которая должна получать новые сообщения из группы MAX, обрабатывать их на backend, доставлять их в Telegram и React-веб-приложение, а также поддерживать web push-уведомления и деплой на Raspberry Pi.

## Приоритет источников

Используй такой порядок:

1. Последнее прямое сообщение пользователя.
2. Этот `AGENTS.md`.
3. Корневой `D:\opencode\AGENTS.md`.
4. `SESSION_CONTEXT.md`, `STAGE_CONTEXT.md`, `SUBSTAGE_CONTEXT.md`, `PLAN.md`.
5. Подтверждённые договорённости и реальная структура проекта.
6. Старые заметки и прошлые черновики.

## Рабочая область проекта

- Работай только внутри папки проекта, пока пользователь явно не сменит рабочую папку.
- Не выноси изменения в другие проекты без прямого запроса.
- Поддерживай простую и последовательную структуру.
- Храни исходники промптов проекта в `prompts/`.
- Держи интеграцию с MAX в отдельном адаптерном слое, чтобы источник можно было заменить без переписывания core-логики.

## Локальный стек контекста

Перед существенной работой по проекту читай:

1. `SESSION_CONTEXT.md`
2. `AGENTS.md`
3. `STAGE_CONTEXT.md`
4. `SUBSTAGE_CONTEXT.md`
5. `PLAN.md` по необходимости
6. релевантные файлы из `docs/`, `src/`, `tests/`, `prompts/`

## Инженерные акценты проекта

- Используй только бесплатный и self-hosted стек.
- Основной узел развёртывания: Raspberry Pi 5.
- Raspberry Pi Zero 2 W рассматривай только как дополнительный bridge, если реально нужен второй сетевой маршрут.
- Не хардкодь токены, chat ID, URL и другие секреты.
- Если внешняя интеграция с MAX ещё не подтверждена, core-сервис всё равно должен оставаться работоспособным с mock- или bridge-адаптером.

## Фразы-триггеры

- По умолчанию проект использует глобальные правила фраз-триггеров из `D:\opencode\WORKFLOW_RULES.md`.
- Базовые фразы: `продолжай`, `статус`, `сделай проверку на баги`.
- Если проекту нужны дополнительные фразы-команды, добавляй их здесь или в `SESSION_CONTEXT.md`.

## Структура проекта

- `docs/` — архитектура, аудит, runbook и deployment-заметки.
- `src/` — backend, frontend и shared-код.
- `tests/` — unit, integration и e2e-проверки.
- `prompts/` — исходники промптов и шаблоны запросов.
