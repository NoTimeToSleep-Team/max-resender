# Постоянный контекст проекта `project-max`

Этот файл хранит постоянные правила, факты и решения проекта `project-max`.

## Идентичность проекта

- Имя проекта: `project-max`
- Основная цель: собрать бесплатную self-hosted систему доставки сообщений из MAX в Telegram и React/PWA-приложение.
- Основная папка работы: `D:\opencode\project-max`

## Постоянные решения

- Основной стек: Node.js + TypeScript для backend, React для frontend, SQLite для хранения и durable queue.
- Основной подход к архитектуре: один core-сервис на Raspberry Pi 5 с модульными адаптерами источников и каналов доставки.
- Базовые ограничения: только бесплатные и self-hosted компоненты, без хардкода секретов и без неподтверждённых заявлений о проверках.
- Ключевые архитектурные принципы: adapter boundary для MAX, idempotency, durable persistence, retry with backoff, минимальная поверхность отказа.

## Правила развития

- Двигаться вертикальными срезами: storage, ingest, delivery, realtime, push, deploy.
- Сначала стабилизировать core-сервис, затем усложнять сетевую схему при реальной необходимости.
- Все секреты, chat ID, URL и ключи держать только в `.env`.
- Держать `PLAN.md` как полный roadmap, а `STAGE_CONTEXT.md` и `SUBSTAGE_CONTEXT.md` как активный рабочий контекст.

## Структура проекта

- `docs/` хранит архитектуру, аудит и runbook.
- `src/` хранит backend, frontend и shared-код.
- `tests/` хранят unit, integration и e2e-проверки.
- `prompts/` хранит исходники промптов и связанные шаблоны запросов.
