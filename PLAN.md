# План проекта `project-max`

## Roadmap

### Этап 01. Foundation

- `v0.01.01` — создать шаблонную структуру проекта, локальный контекст и папку `prompts/`.

### Этап 02. Bootstrap

- `v0.02.01` — провести техаудит, исправить шаблонный drift и выбрать архитектуру.
- `v0.02.02` — развернуть Node.js/TypeScript workspace, базовый HTTP backend, SQLite schema, healthcheck и env validation.
- `v0.02.03` — поднять React/PWA shell с realtime-лентой-заглушкой, тёмной темой и экраном подписки на push.

### Этап 03. Delivery Core

- `v0.03.01` — реализовать durable ingest pipeline, нормализацию сообщений и deduplication.
- `v0.03.02` — добавить очередь доставки, retry with backoff, structured logging и restart recovery.
- `v0.03.03` — подключить Telegram delivery worker и защиту от дублей.

### Этап 04. MAX Integration

- `v0.04.01` — реализовать контракт MAX adapter и локальный mock-адаптер для разработки.
- `v0.04.02` — подключить основной bridge-способ интеграции с MAX.
- `v0.04.03` — добавить резервный режим для сетевого разделения между MAX и Telegram.

### Этап 05. Web UX And Push

- `v0.05.01` — реализовать адаптивную ленту сообщений, статус доставки, пустые и ошибочные состояния.
- `v0.05.02` — добавить service worker, manifest, web push subscriptions и UX-подсказки для iPhone/Android.

### Этап 06. Deployment And Ops

- `v0.06.01` — подготовить `env.example`, install/run scripts, systemd units и reverse proxy конфиг.
- `v0.06.02` — добавить инструкции по обновлению, backup/restore и диагностике.

### Этап 07. Stabilization

- `v0.07.01` — провести сводную проверку, закрыть риски и подготовить первую стабильную поставку для Raspberry Pi.

## Ближайший приоритет

1. `v0.02.02` — workspace, backend foundation и SQLite.
2. `v0.02.03` — frontend shell и PWA-основа.
3. `v0.03.01` — ingest pipeline и хранение сообщений.

## План по сообщениям

Ниже зафиксирован рабочий план по сообщениям в этой сессии.

- Отсчёт начинается со следующего рабочего сообщения после фиксации этого блока.
- В план входит локальная реализация проекта в репозитории.
- В план не входит реальный деплой и проверка на самих `Raspberry Pi`.
- Целевой объём: около `24` рабочих сообщений.
- Резерв: до `30` сообщений на багфиксы, доводку и финальную стабилизацию.

### Уже сделано до этого блока

- изучен контекст рабочей области и проекта;
- проведён стартовый техаудит;
- выбрана архитектура;
- выровнен шаблонный контекст под `project-max`.

### Рабочие сообщения

1. Инициализация workspace: корневой `package.json`, базовая структура `backend/frontend/shared`, TypeScript-конфиги.
2. Настройка общих скриптов, базовых dev-зависимостей и структуры запуска.
3. Поднятие базового backend на `Fastify`.
4. Добавление схемы `.env` и валидации окружения.
5. Подключение `SQLite` и стартовой инициализации БД.
6. Создание таблиц `messages`, `delivery_jobs`, `delivery_attempts`, `push_subscriptions`, `checkpoints`.
7. Реализация `healthcheck` и structured logging.
8. Введение общей схемы сообщения и слоя нормализации входящих данных.
9. Реализация mock-ingest вместо реального `MAX` для локальной разработки.
10. Добавление deduplication и idempotency по ключу сообщения.
11. Реализация durable queue на `SQLite`.
12. Добавление `retry with backoff` и restart recovery.
13. Поднятие `SSE`-потока для realtime-доставки в веб.
14. Инициализация `React + Vite` frontend.
15. Сборка экрана ленты сообщений: автор, время, текст, источник, статус.
16. Добавление адаптивности, тёмной темы и пустых и ошибочных состояний.
17. Подключение PWA-основы: `manifest` и `service worker`.
18. Реализация backend API для `web push subscriptions`.
19. Реализация UI для подписки на push и UX-пояснений для iPhone/Android.
20. Подключение Telegram delivery layer.
21. Форматирование сообщений для Telegram, защита от дублей и статусы доставки.
22. Базовые тесты критического пути backend.
23. Базовые тесты frontend и локальная интеграционная проверка.
24. Документация запуска: `README`, `env.example`, список обязательных секретов и доступов.

### Резервные сообщения

25. Подготовка `systemd`-юнитов и reverse proxy конфигов без реального деплоя на железо.
26. Локальный bug pass по pipeline `ingest -> store -> queue -> web -> telegram`.
27. Локальный bug pass по `push/PWA`.
28. Чистка структуры, мелкий рефакторинг и выравнивание документации.
29. Финальная сводная проверка проекта.
30. Финальный handoff: что подставить пользователю и что проверить уже на реальных `Raspberry Pi`.
