# Стартовый техаудит `project-max`

Дата аудита: `2026-04-21`

## Объём аудита

Проверен весь доступный репозиторий `D:\opencode\project-max`.

## Текущее состояние

- В репозитории есть только контекстные markdown-файлы и стартовые `README.md`.
- В `src/`, `tests/` и `docs/` отсутствуют прикладной код, тесты и эксплуатационные инструкции.
- Нет `package.json`, lockfile, конфигов TypeScript, frontend-сборки, backend-сервера, env-шаблонов и deployment-файлов.
- В `prompts/prompt.md` уже описана реальная цель проекта, но часть локального контекста до этого аудита всё ещё описывала шаблонный `project1`.

## Пробелы

- Нет backend-слоя для ingestion, хранения, очередей, retry и доставки.
- Нет frontend-слоя для realtime-ленты, PWA и push-уведомлений.
- Нет MAX adapter / bridge abstraction.
- Нет интеграции с Telegram Bot API.
- Нет базы данных, схемы миграций и restart recovery.
- Нет `env.example`, валидации окружения и deployment-инструкций для Raspberry Pi.
- Нет healthcheck, structured logging и базовых автоматических проверок.

## Риски

- Доступность официального API MAX не подтверждена в текущем репозитории, поэтому интеграцию нельзя жёстко связывать с одним протоколом.
- Telegram может быть недоступен по тому же маршруту, по которому доступен MAX, поэтому сетевую схему нужно держать заменяемой.
- Web push на iPhone зависит от ограничений iOS и установки PWA на домашний экран.
- Репозиторий стартует с нуля, поэтому важно сначала выбрать стек, который реально поднимается и поддерживается на Raspberry Pi.

## Вывод

Репозиторий пока находится в состоянии шаблона и не готов к запуску. Следующий практический шаг: собрать минимальный production-like каркас на Node.js/TypeScript + React + SQLite, где core-сервис работает даже до окончательного подтверждения интеграции с MAX.
